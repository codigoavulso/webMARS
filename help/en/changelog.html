v0.3.82026-03-10

Runtime limits, examples manifest hardening & WASM-ready core bootstrap

  • Added settings dialog Settings > Uso máximo de Memória... to configure two runtime caps in a single prompt: memory budget (GB) and max backsteps.
  • New persisted preferences: maxMemoryGb (default 2) and maxBacksteps (default 100), wired directly into engine settings.
  • Engine memory writes now enforce maxMemoryBytes; over-limit allocations raise a controlled runtime error instead of runaway growth.
  • Backstep history pruning now obeys dynamic limits during preference changes and supports zero-history mode.
  • Example loading now trusts only examples/examples.json; removed server-dependent directory autodiscovery to guarantee deterministic menus on all hosts.
  • Switched runtime engine bootstrap to createMarsEngine() factory (JS fallback by default, WASM selectable via backend setting).
  • Introduced 00-core-wasm-bridge.js with window.WebMarsWasmCore API (init, registerFactory, createEngineSync) to support future C++/WebAssembly core migration without refactoring UI/runtime modules.
  • Added no-interaction rendering mode in max speed: UI highlights and auto-scroll are disabled while running at “Run speed at max (no interaction)” to reduce layout thrash and keep the browser responsive.
  • Run loop tuning for responsiveness: smaller unlimited batches, tighter cooperative time budget, and throttled UI sync in non-interactive mode.
  • Initial C++ hotpath module scaffold added (web/wasm/core_hotpath.cpp) plus browser loader 00-core-wasm-hotpath.js and build script node wasm/build-wasm-core.mjs (Emscripten).
  • Updated bootstrap module order in app.bundle.js to load WASM bridge before UI/runtime.
v0.3.72026-03-07

Documentation, onboarding & final parity

  • First‑visit “About” dialog and improved example discovery (nested directories).
  • Full support for all MARS 4.5 pseudo‑instructions and exception codes.
  • Memory configuration presets (MEMORY_CONFIG_PRESETS) and dynamic exception handler address.
  • Core feature parity achieved: all syscalls, coprocessors, macros, and the complete MIPS‑32 instruction set (excluding some privileged).
v0.3.62025-12-18

Performance & UI polishing

  • Run loop now uses requestAnimationFrame for batched DOM updates, reducing jank.
  • Snapshot diffing optimized: only changed registers/memory are re‑rendered.
  • “Unsaved changes” guard on page close (beforeunload) and tab refresh.
  • Workspace session automatically persisted to localStorage (SESSION_STORAGE_KEY).

Help system, examples & settings dialogs

  • createHelpSystem() provides tabbed documentation (intro, syscalls, license) with embedded HTML/PDF.
  • Examples menu dynamically loaded from examples.json and directory listing.
  • Settings dialogs via dialogSystem for editor preferences, highlighting, memory presets.
  • Preference store (loadPreferences/savePreferences) with toggles for branching, warnings as errors, etc.
v0.3.42025-02-22

Coprocessor 0/1 & exception handling

  • Full COP0 register set (vaddr, status, cause, epc) and COP1 floating‑point registers.
  • FPU instructions: add.s, mul.d, cvt.w.s, movf, condition flags.
  • raiseException() injects exceptions and redirects to handler (if present) or halts.
  • Added eret instruction and exception handler address configuration.
v0.3.32024-09-18

Data segment navigation & memory views

  • createExecutePane() now renders a paginated data table with base address selection (.data, heap, stack, $gp).
  • Memory words cached in memoryWords map for fast look‑up.
  • Navigation buttons and ASCII/hex/decimal formatting options.
  • Automatic scroll to last written address (lastMemoryWriteAddress).
v0.3.22024-04-10

Run speed control & cooperative loop

  • Speed slider with RUN_SPEED_TABLE (0.05 inst/s to unlimited).
  • Cooperative run loop runLoopTick() with time budgeting, step carrying, and UI sync every 66 ms.
  • Play (go), pause, stop commands with proper cancellation.
  • Interactive mode shows execution messages, fast mode batches up to 1400 steps.
v0.3.12023-11-30

Breakpoints, stepping & register views

  • Breakpoint toggle via toggleBreakpoint() and checkbox in text segment table.
  • Step/backstep implementation: step() saves snapshots, backstep() restores previous state.
  • createRegistersPane() shows integer registers, PC, and highlights changed values.
  • Added computeSnapshotDiff() for efficient UI updates of registers and memory.

MDI window system & layout persistence

  • createWindowManager() provides draggable, resizable, minimizable windows with z‑order stacking.
  • Default desktop with three windows: Main (editor/execute), Registers, Messages.

v0.2.22022-12-05

Syscall I/O and virtual file system

  • Full executeSyscall() with support for print (1–4), read (5–8), sbrk (9), exit (10), file operations (13–16).
  • In‑memory virtual file system (setVirtualFileBytes()) persisted to localStorage (VFS_STORAGE_KEY).
  • Run I/O panel (createMessagesPane()) with input queue and asynchronous reading via consumeInput().
  • Implemented MIDI syscalls (31,33) using Web Audio API (playMidiTone).
v0.2.12022-07-22

Pseudo‑instructions & macro support

  • Extended expandPseudo() to cover all MARS pseudo‑ops: abs, neg, ble, bgt, rol, mul, etc.
  • Implemented .macro/.end_macro parsing with argument substitution and unique label generation.
  • Added .eqv directive handling and applyEqv() for text substitution.
v0.2.02022-03-10

Multi‑file editing & syntax highlighting

  • setupEditor() creates a full‑featured code editor with gutter, line numbers, and real‑time highlighting.
  • Syntax tokens: registers (editor-token-register), directives, labels, comments, strings.
  • Introduced createStore() for central state (files, active file, preferences).
  • File tabs with dirty indicators and basic file operations (new, open, save).
v0.1.22021-08-20

Assembler foundation & directives

  • Two‑pass assembler inside assemble(): first pass collects labels, second pass generates binary.
  • Support for .text, .data, .word, .byte directives via processDataDirective().
  • preprocessSourceLines() strips comments and handles simple .include.
  • First version of expandPseudo() for nop, move, li (using addiu/ori).
v0.1.12021-02-15

Core execution engine

  • Implemented executeInstruction() for R‑type (add, sub, and, or, slt) and I‑type (addi, lw, sw) instructions.
  • Added memory access methods readByte(), writeWord() with alignment checks.
  • Introduced forceZeroRegister() to keep $zero at 0 after every instruction.
  • Basic program counter flow and halt detection.

  • Experimental tokenizeStatement() and parseImmediate() for basic instruction recognition.
