v__WEBMARS_APP_VERSION__2026-03-20

Runtime and Startup Hardening

  • The startup loader now waits for the real webmars:tools-ready state instead of disappearing after manifest load or timeout-based partial boot.
  • Tool bootstrap now falls back to a complete embedded manifest, avoiding the previous collapse to a single available tool when tools.json could not be loaded.
  • Browser-storage metadata now preserves full timestamp precision; updatedAt values are no longer truncated through 32-bit coercion.
  • Runtime/bootstrap dependencies were tightened so unreachable inline fallbacks no longer shadow the active code path.

Help System and Embedded Documents

  • The MIPS reference PDF is now wired back into the Help menu and opens through the embedded document viewer instead of a broken external path.
  • Local serving of PDF assets now uses application/pdf plus inline disposition, allowing in-UI viewing instead of forced download flows.
  • External links, mailto: targets, and explicit new-tab links inside Help/About content now open externally instead of being forced into a blank iframe.
  • The document viewer keeps the requested document title instead of overwriting it with a generic label.

Codebase Cleanup

  • Removed the old ES-module frontend entrypoints under assets/js/app.js, core/, ui/, config/, and state/ that were no longer part of the live bootstrap.
  • Removed the obsolete Help bridge layer and connected the active runtime directly to the modern help system.
  • Pruned dead Mini-C, file-kind, store, and settings fallbacks that could no longer be reached in the ordered bundle bootstrap.

  • Hardened runtime behavior against UI render exceptions that could leave controls disabled.
  • Normalized Mars Messages and Run I/O output emission to reduce excessive blank lines and improve log density.

  • Menu/dialog behavior was further aligned with Java MARS interaction patterns.
