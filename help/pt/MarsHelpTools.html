MARS - Simulador de montagem e tempo de execução de MIPS

Lançamento 4.5

Agosto de 2014

Capacidade legal: Ferramentas de plug-in

Começando com o Release 2.0, o MARS é capaz de executar externamente desenvolvido software que interage com um programa MIPS em execução e sistema MIPS recursos. Os requisitos para tal programa são:
  1. implementa o mars.tools.MarsTool interface.
  2. Faz parte do mars.tools pacote.
  3. Ele compila de forma limpa em um arquivo ".class", armazenado no mars/tools diretório.
O MARS detectará todas as ferramentas de qualificação após a inicialização e as incluirá em seu menu Ferramentas. Quando o item de menu de uma ferramenta é selecionado, uma instância dele será criada usando seu argumento sem argumento construtor e sua action() método será invocado. Se nenhuma ferramenta de qualificação for encontrada na inicialização do MARS, o menu Ferramentas não será aparecem.

Para usar essa ferramenta, carregue e monte um programa de interesse MIPS, selecione a ferramenta desejada no menu Ferramentas. A janela da ferramenta será aberta e, dependendo de como ela está escrita, ela será precisa estar "conectado" ao programa MIPS clicando em um botão ou já estará conectado. Corrida o programa MIPS como você faria normalmente, para iniciar a interação da ferramenta com o programa de execução.

Começando com Release 3.2, a classe abstrata mars.tools.AbstractMarsToolAndApplication está incluído na distribuição MARS para fornecer uma estrutura substancial para implementar sua própria ferramenta MARS. Uma subclasse que a estende por implementar pelo menos seus dois métodos abstratos pode ser executado não apenas a partir do menu Ferramentas, mas também como uma aplicação independente que utiliza o montador e simulador MARS em segundo plano.

Várias Ferramentas desenvolvidas por subclassificação AbstractMarsToolAndApplication estão incluídos com o MARS: uma Introdução às Ferramentas, um Simulador de Cache de Dados, um Visualizador de Referência de Memória, e uma ferramenta Ponto Flutuante. O último é bastante útil mesmo quando não está conectado a um programa MIPS porque exibe representações binárias, hexadecimais e decimais para um ponto flutuante de 32 bits valor; quando qualquer um deles é modificado os outros dois são atualizados também.

A versão 3.5 inclui novas ferramentas, mais notavelmente um simulador de teclado e exibição que permite para realizar E/S mapeada por memória (MMIO) utilizando técnicas pesquisadas e orientadas por interrupções conforme descrito em várias referências. Clique no botão Ajuda para mais detalhes.

Se você deseja desenvolver sua própria ferramenta MARS, primeiro precisará extrair a distribuição MARS a partir de seu arquivo JAR se você ainda não o fez. Todas as ferramentas do MARS devem ser armazenado no mars/tools diretório.

Siga o link Materiais do Tutorial na página inicial do MARS para encontrar um tutorial que abrange desenvolvimento de Ferramentas MARS.

Capacidade legal: estendendo o conjunto de syscall ou reatribuindo números de syscall

Começando com Release 3.1, chamadas de sistema (syscall instrução) são implementadas usando uma técnica semelhante à para as ferramentas. Isso permite que qualquer pessoa adicione uma nova syscall definindo uma nova classe que atenda a esses requisitos:
  1. implementa o mars.mips.instructions.syscalls.Syscall interface, ou estende o mars.mips.instructions.syscalls.AbstractSyscall classe (que fornece implementações padrão de tudo, exceto o simulate() método).
  2. Faz parte do mars.mips.instructions.syscalls pacote.
  3. Ele compila de forma limpa em um arquivo ".class", armazenado no mars/mips/instructions/syscalls diretório.
O MARS detectará todas as classes de syscall qualificadas após a inicialização e o simulador de tempo de execução invocá-los-á quando o syscall instrução é simulada e registra $v0 contém o número de serviço de inteiro correspondente.

Syscalls e syscall number assignments em MARs correspondem aos de SPIM para syscalls 1 a 17. No entanto, se você deseja alterar as atribuições de número de syscall, você pode fazê-lo editando o Syscall.properties arquivo incluído na versão (isso requer extração do JAR arquivo).

Siga o link Materiais do Tutorial na página inicial do MARS para encontrar um tutorial que abrange desenvolvimento de chamadas de sistema.

Capacidade legal: estendendo o conjunto de instruções

Você pode adicionar pseudo-instruções personalizadas ao conjunto de instruções do MIPS editando o PseudoOps.txt arquivo incluído na distribuição MARS. Instrução formatos de especificação são explicados no próprio arquivo. A especificação de uma pseudo-instrução é de uma linha de comprimento. consiste em um exemplo da instrução, construída usando símbolos de especificação de instruções disponíveis, seguidos por uma guia separada lista das instruções básicas do MIPS para as quais ele se expandirá. Cada um é um modelo de instrução construídos usando símbolos de especificação de instruções combinados com modelo especial símbolos de especificação. A última autorização de substituição no programa tempo de montagem de operandos do programa do usuário para o expandido pseudo-instrução.

PseudoOps.txt é lido e processado na inicialização do MARS, e as mensagens de erro serão ser produzido se uma especificação não estiver corretamente formatada. Observe que se você deseja editá-lo você primeiro tem que extraí-lo do arquivo JAR.

Siga o link Materiais do Tutorial na página inicial do MARS para encontrar um tutorial que abrange modificação do conjunto de pseudo-instrução.
