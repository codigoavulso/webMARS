Usando o MARS na linha de comando.

O MARS pode ser executado a partir de um intérprete de comando para montar e executar um programa MIPS em uma modo em lote. O formato para executar o MARS a partir na linha de comando é:

java -jar mars.jar [options] program.asm [more files...] [ pa arg1 [more args...]]

Itens em [ ] são opcionais. Opções válidas (não sensíveis a casos, separadas por espaços) são:

OpçãoDescriçãoDesde
amontar apenas, não simular1.0
aenencerrar o MARS com o código de saída inteiro n se ocorrer erro de montagem4.1
asciiexibir memória ou registrar conteúdo interpretado como códigos ASCII. (alternativas são dec e hex)4.1
bbreve - não exibir registro / endereço de memória, juntamente com o conteúdo2.2
dexibir declarações de depuração MARS (de interesse principalmente para desenvolvedor MARS)1.0
dbA ramificação atrasada do MIPS está habilitada.3.3
decexibir memória ou registrar conteúdo em decimal. (alternativas são ascii e hex)2.2
dumpdespeje o conteúdo da memória para arquivar. A opção tem 3 argumentos, por exemplo. dump <segment> <format> <file>. Os segmentos atuais suportados são .text e .data. Também suporta um intervalo de endereços (veja m-n abaixo). Os formatos de dump suportados atuais são Binary, HexText, BinaryText, AsciiText. Veja exemplos abaixo.3.4
hexexibir memória ou registrar conteúdo em hexadecimal - este é o padrão. (alternativas são ascii e dec)2.2
hexibir essa ajuda. Use esta opção por si só e sem nome de arquivo.1.0
iccontagem de instruções de exibição; o número de instruções básicas MIPS 'executadas'4.3
mcdefinir configuração de memória. A opção tem 1 argumento, por exemplo. mc <config>. Argumento <config> é case-sensitive e sua valores possíveis são Default para o espaço padrão de endereço de 32 bits, CompactDataAtZero para um espaço de endereço de 32KB com segmento de dados no endereço 0, ou CompactTextAtZero para um espaço de endereço de 32KB com segmento de texto no endereço 0.3,7
meexibir mensagens MARS para erro padrão em vez de padrão para fora. Permite separar as mensagens MARS da saída do programa MIPS usando o redirecionamento.4.3
ncaviso de direitos autorais não será exibido. Útil se redirecionar ou canalizar a saída do programa.3.5
nppseudo-instruções ou formatos de instrução estendidos não são permitidos.3.0
popção de projeto - irá montar o arquivo especificado e todos os outros arquivos de montagem (*.asm; *.s) em seu diretório.3.1
senterminar MARS com código de saída n se simular (execução) erro ocorre4.1
sminiciar a execução em declaração tendo rór?tulo global 'main' se definido3.8
smcCódigo de Auto Modificação - O programa pode escrever e executar em qualquer segmento de texto ou dados4.4
weavisos de assembler serão considerados erros.3.5
nonde n é uma contagem máxima inteira de etapas de execução para simular. Se 0, negativo ou não especificado, não há máximo.1.0
$ regonde reg é número ou nome (por exemplo, 5, t3, f10) do registo cujo registo conteúdo para exibir no final da execução. O registro de ponto flutuante com número uniforme exibe tanto ponto flutuante quanto duplo. A opção pode ser repetida. OBSERVAÇÃO: Dependendo do shell do seu comando, talvez seja necessário escapar do $, por exemplo. \$t32.2
reg_nameonde reg_name é o nome (por exemplo, t3, f10) do registro cujo conteúdo para exibir no final da execução. O registro de ponto flutuante com número uniforme exibe tanto ponto flutuante quanto duplo. A opção pode ser repetida. $ não é necessária.2.2
m - nendereço de memória variam de m a n cujo conteúdo para exibição no final da corrida. m e n pode ser decimal ou hexadecimal (começa com 0x), m <= n, ambos devem estar no limite da palavra. A opção pode ser repetida.2.2
paargumentos do programa - todos os itens separados pelo espaço restantes são valores de argumento fornecidos ao MIPS programa via $a0 (argc - contagem de argumentos) e $a1 (argv - endereço de array contendo ponteiros para argumento nulo-terminado cordas). A contagem também está no topo da pilha de tempo de execução ($sp), seguida pela opção array.Esta e seus argumentos devem ser os últimos itens no comando!3.5

Exemplo: java -jar mars.jar h
Exibe opções de comando e explicações.

Exemplo: java -jar mars.jar $s0 $s1 0x10010000-0x10010010 fibonacci.asm
Monte e corra fibonacci.asm. No final da execução, exibir o conteúdo dos registros $s0 e $s1, e o conteúdo de locais de memória 0x10010000 até 0x100100100. O conteúdo é exibido em formato hexadecimal.

Exemplo: java -jar mars.jar a fibonacci.asm
Montar fibonacci.asm. Não tenta executar o programa e o código montado não é salvo.

Exemplo: java -jar mars.jar 100000 infinite.asm
Monte e corra infinite.asm para um máximo de 100.000 etapas de execução.

Exemplo: java -jar mars.jar p major.asm
Montar major.asm e todos os outros arquivos no mesmo diretório, vincular o código montado e executar começando com a primeira instrução em major.asm.

Exemplo: java -jar mars.jar major.asm minor.asm sub.asm
Montar e link major.asm, minor.asm e sub.asm. Se for bem sucedida, execução começará com a primeira instrução em major.asm.

Exemplo: java -jar mars.jar a dump .text HexText hexcode.txt fibonacci.asm
Montar fibonacci.asm sem simular (note uso de opção 'a'). No final da montagem, despeje o segmento de texto (código da máquina) para arquivo hexcode.txt em formato de texto hexadecimal com uma instrução por linha.

Exemplo: java -jar mars.jar dump 0x10010000-0x10010020 HexText hexcode.txt fibonacci.asm
Monte e simule fibonacci.asm. No final da simulação, despeje o conteúdo dos endereços 0x1001000 para 0x10010020 para o arquivo hexdata.txt em formato de texto hexadecimal com uma palavra por linha.

Exemplo: java -jar mars.jar t0 process.asm pa counter 10
Monte e corra process.asm com dois valores de argumento de programa, "counter" e "10". Pode recuperar o contagem de argumentos (2) de $a0, e o endereço de uma matriz contendo ponteiros para as strings "count" e "10", de $a1. No fim da execução, exibir o conteúdo do registro $t0.

A capacidade de executar o MARS a partir da linha de comando é útil se você quiser desenvolver scripts (macros) para exercer um determinado programa MIPS em vários cenários ou se você quiser executar uma série de diferentes programas MIPS tais como para fins de classificação.
