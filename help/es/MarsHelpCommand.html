Version 4.5

Agosto de 2014

Uso de MARS desde la linea de comandos.

MARS puede ejecutarse desde un interprete de comandos para ensamblar y ejecutar un programa MIPS en modo por lotes. El formato para ejecutar MARS desde la linea de comandos es:

java -jar mars.jar [options] program.asm [more files...] [ pa arg1 [more args...]]

Los elementos entre [ ] son opcionales. Las opciones validas (sin distinguir mayusculas de minusculas y separadas por espacios) son:

OptionDescripcionDesde
asolo ensamblar, no simular1.0
aenfinalizar MARS con el codigo de salida entero n si se produce un error de ensamblado4.1
asciimostrar el contenido de memoria o de registros interpretado como codigos ASCII (las alternativas son dec y hex)4.1
bmodo resumido - no mostrar la direccion del registro o de la memoria junto con el contenido2.2
dmostrar mensajes de depuracion de MARS (de interes sobre todo para quien desarrolla MARS)1.0
dbactivar MIPS delayed branching.3.3
decmostrar el contenido de memoria o registros en decimal (las alternativas son ascii y hex)2.2
dumpvolcar el contenido de memoria a un archivo. La opcion tiene 3 argumentos, por ejemplo dump <segment> <format> <file>. Los segmentos actualmente soportados son .text y .data. Tambien admite un rango de direcciones (vease m-n mas abajo). Los formatos de volcado soportados actualmente son Binary, HexText, BinaryText, AsciiText. Consulte los ejemplos mas abajo.3.4
hexmostrar el contenido de memoria o registros en hexadecimal; esta es la opcion predeterminada (las alternativas son ascii y dec)2.2
hmostrar esta ayuda. Use esta opcion sola y sin nombre de archivo.1.0
icmostrar el instruction count; el numero de instrucciones basicas MIPS 'ejecutadas'4.3
mcestablecer la configuracion de memoria. La opcion tiene 1 argumento, por ejemplo mc <config>. El argumento <config> distingue mayusculas y minusculas y sus valores posibles son Default para el espacio de direcciones predeterminado de 32 bits, CompactDataAtZero para un espacio de direcciones de 32KB con el segmento de datos en la direccion 0, o CompactTextAtZero para un espacio de direcciones de 32KB con el segmento de texto en la direccion 0.3.7
memostrar los mensajes de MARS en standard err en lugar de standard out. Permite separar los mensajes de MARS de la salida del programa MIPS usando redireccion.4.3
ncno mostrar el aviso de copyright. Resulta util si se redirige o encadena la salida del programa.3.5
npno se permiten pseudo-instrucciones ni formatos extendidos de instruccion.3.0
popcion de proyecto - ensamblara el archivo especificado y todos los demas archivos de ensamblador (*.asm; *.s) de su directorio.3.1
senfinalizar MARS con el codigo de salida n si se produce un error de simulacion (run)4.1
sminiciar la ejecucion en la sentencia con la etiqueta global 'main' si existe3.8
smcSelf Modifying Code - el programa puede escribir y ejecutar tanto en el segmento de texto como en el de datos4.4
welos warnings del ensamblador se consideraran errores.3.5
ndonde n es el numero maximo entero de pasos de ejecucion a simular. Si es 0, negativo o no se especifica, no hay maximo.1.0
$regdonde reg es el numero o nombre (por ejemplo 5, t3, f10) del registro cuyo contenido se mostrara al final de la ejecucion. Un registro flotante par muestra tanto float como double. La opcion puede repetirse. NOTA: segun su shell de comandos, puede que necesite escapar el $, por ejemplo \$t32.2
reg_namedonde reg_name es el nombre (por ejemplo t3, f10) del registro cuyo contenido se mostrara al final de la ejecucion. Un registro flotante par muestra tanto float como double. La opcion puede repetirse. No se requiere $.2.2
m-nrango de direcciones de memoria desde m hasta n cuyo contenido se mostrara al final de la ejecucion. m y n pueden ser decimales o hexadecimales (empiezan por 0x), m <= n, y ambos deben estar alineados a palabra. La opcion puede repetirse.2.2
paargumentos del programa - todos los elementos restantes separados por espacios se proporcionan al programa MIPS mediante $a0 (argc - numero de argumentos) y $a1 (argv - direccion del array que contiene punteros a cadenas terminadas en null que representan los argumentos). El contador tambien aparece en la parte superior de la pila de ejecucion ($sp), seguido por el array. Esta opcion y sus argumentos deben ser los ultimos elementos del comando.3.5

Ejemplo: java -jar mars.jar h
Muestra las opciones de linea de comandos y sus explicaciones.

Ejemplo: java -jar mars.jar $s0 $s1 0x10010000-0x10010010 fibonacci.asm
Ensamblar y ejecutar fibonacci.asm. Al final de la ejecucion, mostrar el contenido de los registros $s0 y $s1, asi como el contenido de las posiciones de memoria 0x10010000 a 0x10010010. El contenido se muestra en formato hexadecimal.

Ejemplo: java -jar mars.jar a fibonacci.asm
Ensamblar fibonacci.asm. No intenta ejecutar el programa y el codigo ensamblado no se guarda.

Ejemplo: java -jar mars.jar 100000 infinite.asm
Ensamblar y ejecutar infinite.asm con un maximo de 100.000 pasos de ejecucion.

Ejemplo: java -jar mars.jar p major.asm
Ensamblar major.asm y todos los demas archivos del mismo directorio, enlazar el codigo ensamblado y ejecutar comenzando por la primera instruccion de major.asm.

Ejemplo: java -jar mars.jar major.asm minor.asm sub.asm
Ensamblar y enlazar major.asm, minor.asm y sub.asm. Si todo sale bien, la ejecucion comenzara en la primera instruccion de major.asm.

Ejemplo: java -jar mars.jar a dump .text HexText hexcode.txt fibonacci.asm
Ensamblar fibonacci.asm sin simular (observe el uso de la opcion 'a'). Al final del ensamblado, volcar el segmento de texto (codigo maquina) al archivo hexcode.txt en formato de texto hexadecimal, con una instruccion por linea.

Ejemplo: java -jar mars.jar dump 0x10010000-0x10010020 HexText hexcode.txt fibonacci.asm
Ensamblar y simular fibonacci.asm. Al final de la simulacion, volcar el contenido de las direcciones 0x1001000 a 0x10010020 al archivo hexdata.txt en formato de texto hexadecimal, con una palabra por linea.

Ejemplo: java -jar mars.jar t0 process.asm pa counter 10
Ensamblar y ejecutar process.asm con dos argumentos de programa, "counter" y "10". El programa puede recuperar el numero de argumentos (2) desde $a0, y la direccion del array que contiene punteros a las cadenas "count" y "10", desde $a1. Al final de la ejecucion, mostrar el contenido del registro $t0.

La posibilidad de ejecutar MARS desde la linea de comandos es util si desea desarrollar scripts (macros) para probar un programa MIPS concreto bajo varios escenarios, o si quiere ejecutar distintos programas MIPS, por ejemplo con fines de evaluacion.
