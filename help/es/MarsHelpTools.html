MARS - Mips Assembly and Runtime Simulator

Version 4.5

Agosto de 2014

Capacidad interesante: herramientas plug-in

A partir de la version 2.0, MARS es capaz de ejecutar software desarrollado externamente que interactua con un programa MIPS en ejecucion y con los recursos del sistema MIPS. Los requisitos para ese tipo de programa son:
  1. Implementar la interfaz mars.tools.MarsTool.
  2. Formar parte del paquete mars.tools.
  3. Compilar correctamente en un archivo ".class", almacenado en el directorio mars/tools.
MARS detectara todas las herramientas validas al iniciarse y las incluira en su menu Tools. Cuando se seleccione la opcion de menu de una herramienta, se creara una instancia mediante su constructor sin argumentos y se invocara su metodo action(). Si al iniciar MARS no se encuentra ninguna herramienta valida, el menu Tools no aparecera.

Para utilizar una de estas herramientas, cargue y ensamble un programa MIPS de interes y despues seleccione la herramienta deseada en el menu Tools. Se abrira la ventana de la herramienta y, dependiendo de como este implementada, o bien necesitara "conectarse" al programa MIPS pulsando un boton, o bien ya estara conectada. Ejecute el programa MIPS como lo haria normalmente para iniciar la interaccion de la herramienta con el programa en ejecucion.

A partir de la version 3.2, la clase abstracta mars.tools.AbstractMarsToolAndApplication se incluye en la distribucion de MARS para proporcionar una base importante para implementar su propia herramienta MARS. Una subclase que la extienda e implemente al menos sus dos metodos abstractos puede ejecutarse no solo desde el menu Tools, sino tambien como una aplicacion independiente que use el ensamblador y el simulador de MARS en segundo plano.

Varias herramientas desarrolladas como subclases de AbstractMarsToolAndApplication se incluyen con MARS: una Introduccion a Tools, un Data Cache Simulator, un Memory Reference Visualizer y una herramienta de coma flotante. La ultima es especialmente util incluso cuando no esta conectada a un programa MIPS porque muestra las representaciones binaria, hexadecimal y decimal de un valor de coma flotante de 32 bits; cuando cualquiera de ellas se modifica, las otras dos se actualizan tambien.

La version 3.5 incluye nuevas herramientas, sobre todo un simulador de teclado y pantalla que permite realizar E/S mapeada en memoria (MMIO) usando tecnicas por sondeo e impulsadas por interrupciones, tal como se describe en varias referencias. Pulse su boton Help para obtener mas detalles.

Si desea desarrollar su propia herramienta MARS, primero tendra que extraer la distribucion de MARS de su archivo JAR si todavia no lo ha hecho. Todas las herramientas MARS deben almacenarse en el directorio mars/tools.

Siga el enlace Tutorial Materials en la pagina principal de MARS para encontrar un tutorial que cubre el desarrollo de herramientas MARS.

Capacidad interesante: ampliar el conjunto de syscalls o reasignar sus numeros

A partir de la version 3.1, las system calls (instruccion syscall) se implementan usando una tecnica similar a la de las herramientas. Esto permite a cualquiera anadir una nueva syscall definiendo una nueva clase que cumpla estos requisitos:
  1. Implementar la interfaz mars.mips.instructions.syscalls.Syscall o extender la clase mars.mips.instructions.syscalls.AbstractSyscall (que proporciona implementaciones por defecto de todo excepto del metodo simulate()).
  2. Formar parte del paquete mars.mips.instructions.syscalls.
  3. Compilar correctamente en un archivo ".class", almacenado en el directorio mars/mips/instructions/syscalls.
MARS detectara todas las clases de syscall validas al arrancar y el simulador de ejecucion las invocara cuando se simule la instruccion syscall y el registro $v0 contenga el numero entero de servicio correspondiente.

Las syscalls y la asignacion de sus numeros en MARS coinciden con las de SPIM para las syscalls 1 a 17. Sin embargo, si desea cambiar la asignacion de numeros de syscall, puede hacerlo editando el archivo Syscall.properties incluido en la distribucion (para ello hay que extraerlo del archivo JAR).

Siga el enlace Tutorial Materials en la pagina principal de MARS para encontrar un tutorial que cubre el desarrollo de system calls.

Capacidad interesante: ampliar el conjunto de instrucciones

Puede anadir pseudo-instrucciones personalizadas al conjunto de instrucciones MIPS editando el archivo PseudoOps.txt incluido en la distribucion de MARS. Los formatos de especificacion de instrucciones se explican en el propio archivo. La especificacion de una pseudo-instruccion ocupa una sola linea. Consiste en un ejemplo de la instruccion, construido mediante los simbolos disponibles de especificacion de instrucciones, seguido de una lista separada por tabuladores con las instrucciones MIPS basicas en las que se expandira. Cada una es una plantilla de instruccion construida usando simbolos de especificacion de instrucciones combinados con simbolos especiales de plantilla. Estos ultimos permiten sustituir en tiempo de ensamblado los operandos del programa del usuario en la pseudo-instruccion expandida.

PseudoOps.txt se lee y procesa al iniciar MARS, y se produciran mensajes de error si una especificacion no tiene el formato correcto. Tenga en cuenta que, si desea editarlo, primero tendra que extraerlo del archivo JAR.

Siga el enlace Tutorial Materials en la pagina principal de MARS para encontrar un tutorial que cubre la modificacion del conjunto de pseudo-instrucciones.
