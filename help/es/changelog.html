v__WEBMARS_APP_VERSION__2026-03-19

Backend Cloud y Autenticación

  • Se añadió selección configurable del backend cloud, con enrutado automático al backend local en desarrollo y a backmars.nfiles.top/api en producción cuando no hay servidor personalizado configurado.
  • Se integraron en la UI los flujos de login, registro y creación de usuario cloud, con soporte backend listo para producción y feedback operativo más claro.
  • Se separó la autenticación de la sincronización de proyectos para que login/registro terminen inmediatamente mientras la sync continúa en segundo plano.
  • Se mejoraron los diálogos de login/registro con estado dentro de la propia ventana, tiempo transcurrido, cancelación más segura y mensajes explícitos de actividad cloud en Mars Messages.

Mejoras en el Flujo de Project y Editor

  • El flujo principal de autoría se consolidó alrededor de Project, Editor y Execute, manteniendo la disponibilidad de acciones C0/Assembly según el tipo del archivo activo.
  • El árbol de proyecto ganó defaults más útiles de expandir/colapsar, reglas de apertura en read-only para libs/ y archivos de otros proyectos, selección múltiple, metadatos/estado e indicadores de sincronización.
  • Las tabs del editor ahora restauran correctamente sesiones read-only e incluyen menú contextual con Close, Save y Save and Close.
  • Las operaciones de guardado se simplificaron a Save Active File y Save Project, con persistencia en el navegador y sincronización cloud cuando corresponde.

Refinamiento de Almacenamiento, Mini-C y UI

  • La cuota de almacenamiento del usuario en el navegador se elevó a 1 MB, se movió a configuración central y ahora se aplica de forma consistente.
  • El flujo de salida de Mini-C se refinó con opción para suprimir la ventana de output, apertura directa del ASM generado y cierre automático tras abrir ASM.
  • Settings cloud/servidor, traducciones y comportamiento de diálogos se ampliaron para soportar el nuevo modelo de despliegue y el flujo backend de producción.
